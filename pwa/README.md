# 証憑ポケット（PWA版）— スマホにアプリとして入れる手順

このフォルダ一式をWebに置く（ホスティングする）と、スマホのホーム画面に
アプリとして追加でき、カメラ撮影・PDF保存・オフライン起動が使えます。
データ（PDF・記録）はすべて端末内（IndexedDB）に保存されます。

## 1. ホスティング（どちらか。無料・5分）

### 方法A：Netlify Drop（いちばん簡単・アカウント不要でも可）
1. パソコンかスマホで https://app.netlify.com/drop を開く
2. このフォルダ（zipを展開したもの）を画面にドラッグ＆ドロップ
3. 発行されたURL（https://～.netlify.app）をスマホに送る

### 方法B：GitHub Pages
1. GitHubで新しいリポジトリを作成し、このフォルダの中身をアップロード
2. Settings → Pages → Branch: main / root を選んで保存
3. 表示されたURLを開く

※ https のURLで開くことが必須です（カメラとアプリ化の条件）。

## 2. スマホでアプリとして追加

- **iPhone（Safari）**：URLを開く → 共有ボタン（□↑）→「ホーム画面に追加」
- **Android（Chrome）**：URLを開く → メニュー（⋮）→「アプリをインストール」
  （または「ホーム画面に追加」）

以後はホーム画面のアイコンから、全画面のアプリとして起動します。

## 3. 自動読み取り（AI-OCR）を使う場合のみ

1. https://console.anthropic.com でAPIキーを発行
2. アプリ右上の ⚙（設定）→ APIキーを貼り付けて保存

- キーは端末内にのみ保存されます。**共用端末には登録しないでください。**
- キー未登録でも、撮影・PDF保存・一覧・編集・削除・集計・CSV出力は
  すべて使えます（項目は手入力）。

## 4. 注意

- データは端末のブラウザ内に保存されます。端末変更・ブラウザのデータ消去で
  失われるため、重要な書類はPDFダウンロードやCSVで定期的に控えを取ってください。
- 電子帳簿保存法の法定要件（タイムスタンプ又は事務処理規程、解像度要件など）を
  完全に満たすには、社内規程の整備やJIIMA認証製品の利用をご検討ください。

## ファイル構成
- index.html … アプリ本体
- manifest.webmanifest … アプリ情報（名前・アイコン）
- sw.js … オフライン用サービスワーカー
- icon-192.png / icon-512.png / icon-512-maskable.png … アプリアイコン


## 5. 今回の最終版での変更点

- 自動読み取りを強化：画像の自動コントラスト補正（感熱紙の薄い印字対応）、
  応答上限の拡大（明細が多いレシートで途中で失敗する問題を解消）、
  和暦の日付・支払方法の自動判定、混雑時の自動リトライ、読み取り中の中止ボタン
- ⚙設定に「接続テスト」を追加：APIキーが正しいかワンタップで確認できます
- 編集シートから全項目（税率内訳・但し書き・勘定科目・支払方法）を修正可能に
- 保存時に「内訳合計＞取引金額」の読み取り誤りを検知して確認を表示
- アプリ更新が確実に反映される方式に変更（本体はネット優先、圏外時はキャッシュ起動）。
  すでにホーム画面に追加済みの場合も、新しいファイルを同じURLに上書きアップロード
  すれば、次回起動時（またはその次）に自動で最新版になります。
